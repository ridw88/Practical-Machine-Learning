Introduction

 

The purpose of this submission is to elaborate on the processes in making the machine learning prediction to predict how well certain individual doing a weight lifting exercises. In order to make a prediction, datasets from project Human Activity Recognition (HAR) is used. Using these datasets with proper machine learning algorithm, the author expects that the algorithm able to make a good predictions on how well the individual does the exercise by classifying it to five different categories namely: exactly according to the specification (Class A), throwing the elbows to the front (Class B), lifting the dumbbell only halfway (Class C), lowering the dumbbell only halfway (Class D) and throwing the hips to the front (Class E). A complete description about the processes including process in generating the model, cross validation, and out of sample error will be discussed in later part.

 

Model building

In building the model, we must be aware of datasets available for the purpose of this project. To name it, in the context of this submission, below datasets were available

1.      pml-training.csv Datasets for training and cross validating

2.      pml-testing.csv Datasets for final testing

The author quickly recognized that some datasets cleaning is required before proceed with choosing the right algorithm. Given the coherency in column for both training and testing set, any datasets cleaning applicable to training set is also applicable to testing set. The processes are as follow:

1.      Eliminate non numeric column (i.e. user name, cvd timestamp, and new_window)

2.      Eliminate column with blank or NA

3.      Eliminate column that has almost zero variance. This is achieved by calling function nearZeroVar

 To know the data, it is useful to know the correlation among the datasets that finally being used. Among 1485 possible combinations only 19 of them are highly correlated (i.e. correlation > 0.8) which suggest that assuming all readings/variables are important, we were dealing with non-linear model. Figure 1 shows the illustration of non-linear relationship and difficulties in finding pattern among different classes. One can obviously experimenting with linear model and further reduce datasets to capture only the most important datasets. However due to limited time and above considerations, the author has decided not to pursue linear model, instead choosing versatile random forest algorithm to proceed with the machine learning algorithm building. No preprocessing was done. Although PCA might be a good candidates, however given most of the variables are not highly correlated, the author thought the reduction in predictors might not be significant by doing PCA and opt to forego this option.

Cross validation and out of sample error

To do a cross validation, the author splits the training datasets into subset of training and testing datasets with proportion 60/40. As an alternative K-fold algorithm could be used but since random forest model is actually subsamples the covariates several times and averages the multiple tree results, it is essentially almost the same as doing K-fold. Further random forest is an expensive algorithm, thus given considerations, the author has decided to go for simple 60/40 training data splits to have cross validation done. Hence following configuration was achieved

To facilitate the training time, the author made use recommended article listed in (2) to do a parallel processing using foreach in combination with randomforest package. This decreases the processing time at minimum by a multiple of core the processor have. On top of that, users could play around with number of tree deemed reasonable to get the intended accuracy. The logic behind choosing the number of trees is beyond the scope of this submission, thus the author chosen arbitrary number and decided the recommended value of 1000 trees (250 X 4) was enough to achieve intended accuracy. Giving this configurations, the author was able to predict the training and cross validation with 100% and 99.85%.

Future recommendation

Given more time, the author will suggest a comparison with different model including using the possibility of using the predictor on the final model based on another trained model. More rigorous data identification to help identify the most important covariates and more rigorous cross validation will also ensure the final model to be more robust, less risk of overfitting, and execute faster in general.
